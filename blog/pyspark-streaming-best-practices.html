PySpark Streaming Best Practices

Lessons from building production-grade PySpark Structured Streaming pipelines — from exactly-once semantics to operational monitoring.

1. Design for Fault Tolerance

  • Always enable checkpointing to recover state and track offsets across restarts.
  • Use idempotent sinks so replayed writes do not corrupt downstream data.
  • Write a foreachBatch sink when you need fine-grained transactional control.
Tip: Store checkpoints on a durable, distributed location such as S3, ADLS, or a shared mount — not on a local driver.

2. Control Late Arrivals with Watermarks

Late data is normal in streaming. Define an event-time watermark so Spark can drop data that arrives after the threshold, keeping state bounded and pipelines stable.

Example: Use withWatermark("eventTime", "10 minutes") before aggregations to cap state growth.

3. Enforce Schema and Validate Data

  • Start with a strict schema instead of inferring from JSON, so upstream changes fail early.
  • Use a dead-letter queue or bad-record handler to quarantine malformed events.
  • Track data quality metrics as part of the pipeline run.

4. Deduplicate Intelligently

Use watermark + dropDuplicates for simple dedup and MERGE into Delta Lake for more complex upserts. Keep deduplication keys small and indexed where possible.

Tip: For exactly-once end-to-end guarantees, combine idempotent Delta Lake merges with deterministic reprocessing logic.

5. Right-Size Resources

  • Match the number of partitions to source throughput and available cores.
  • Use coalesce or repartition before writing to avoid small files in Delta Lake.
  • Enable adaptive query execution (AQE) and monitor shuffle sizes.

6. Monitor and Alert

  • Log structured events such as batch duration, input rate, and offsets processed.
  • Track watermark lag and state store size in Spark UI or via streaming listener metrics.
  • Set alerts on processing time exceeding trigger interval or consumer lag spiking.

7. Keep Business Logic Clean

  • Separate pure transformation functions from streaming orchestration code for easy unit testing.
  • Parameterize pipeline configuration (source, trigger, output) from environment variables or YAML.
  • Write pytest suites with in-memory Spark fixtures before deploying to production.
