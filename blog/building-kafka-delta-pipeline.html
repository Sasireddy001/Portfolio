Building a Kafka → Delta Pipeline

A hands-on guide to building a production-grade streaming data pipeline with Apache Kafka, PySpark Structured Streaming, and Delta Lake.

Architecture Overview

Kafka Topic → Spark Structured Streaming → Transformations → Delta Lake Sink

This pattern is the core of modern lakehouse streaming. Kafka acts as the durable, scalable event bus. Spark reads micro-batches, applies business logic, and writes to Delta Lake with exactly-once guarantees.

Step 1: Kafka Source Configuration

  • Use subscribe or subscribePattern to define the Kafka topics.
  • Set startingOffsets to latest for fresh starts or earliest for backfills.
  • Specify failOnDataLoss=false only when you understand the trade-off.
  • Use SSL / SASL configs for enterprise security.
Tip: Keep the number of Spark partitions equal to Kafka topic partitions for best throughput and parallelism.

Step 2: Schema & Parsing

  • Define a strict schema in PySpark instead of inferring from JSON strings.
  • Parse the Kafka value field from binary to string/JSON first, then apply the schema.
  • Route malformed records to a dead-letter stream or log-and-drop path.

Step 3: Transformations & State

  • Use withWatermark on event time before aggregations to bound state.
  • Deduplicate with dropDuplicates using a composite key (event ID + source timestamp).
  • Keep transformation functions pure and unit-testable with pytest and in-memory Spark.

Step 4: Delta Lake Sink

  • Write in append mode for immutable streams and complete mode for aggregations.
  • Use a checkpointLocation for reliable offset tracking across restarts.
  • For idempotent upserts, use foreachBatch with a Delta Lake MERGE.
  • Partition by logical keys (date, region) to optimize downstream reads.
Tip: A robust pipeline combines checkpointing for offset recovery with Delta Lake merge keys for idempotent writes.

Step 5: Operational Concerns

  • Enable structured streaming listener metrics to track input rate, batch duration, and state store size.
  • Right-size cluster cores and executor memory for the trigger interval.
  • Handle skewed keys with salting or adaptive query execution where applicable.
  • Keep secrets out of code — use Vault or environment-backed configuration.

Putting It Together

A production Kafka → Delta pipeline is more than code. It is a combination of schema discipline, exactly-once semantics, state management, and operational observability. Get these right, and the system scales from thousands to millions of events per hour.
